    MonaDev - Plateforme de création d'événement dans des lieux de Patrimoine Culturel

    Contexte et Enjeux

    Dans le cadre d'un projet semestriel à l'IUT2 de Grenoble, notre équipe de 7 étudiants a développé Mona, une plateforme web innovante inspirée du modèle Airbnb pour valoriser le patrimoine culturel français. Le défi consistait à créer une solution permettant :

    • La découverte interactive de 45 000+ monuments historiques
    • La réservation d'expériences culturelles (chasses au trésor, événements)
    • La gestion simplifiée pour propriétaires de monuments

    Le projet s'inscrivait dans une démarche pluridisciplinaire intégrant développement web, gestion de projet, analyse de données et cybersécurité.

    Organisation et Méthodologie

    Nous avons adopté une gestion agile répartie sur 4 phases :

    Phase 1

    Analyse des besoins
    Personas
    Étude de faisabilité

    Phase 2

    Modélisation UML
    Architecture technique
    ETL des données

    Phase 3

    Développement intensif
    Tests utilisateurs

    Outils clés : Notion pour la gestion de projet, GitLab avec bot Discord pour la coordination, et Pair Programming pour assurer la qualité du code.

    Réalisation Technique

    Frontend

    • ReactJS avec architecture composants
    • Palette CSS harmonisée
    • Système de routing dynamique

    Backend

    • API REST custom en PHP
    • PostgreSQL pour la base de données
    • Talend pour l'ETL des données Mérimée

    Infrastructure

    • Déploiement sur VM IUT
    • HTTPS avec certificat auto-signé
    • Firewall et fail2ban

    Innovation majeure : système ETL croisant les bases Mérimée (data.gouv.fr) et OpenStreetMap pour enrichir automatiquement les fiches monuments avec des adresses précises.

    Défis et Solutions

    Cyberattaque IONOS

    Notre serveur initial a été compromis par un ransomware. Solution : migration rapide vers les VM IUT avec restauration depuis les sauvegardes SQL.

    Intégration ReactJS

    Difficultés d'apprentissage pour l'équipe. Solution : TP préparatoire et composants standardisés.

    Qualité des données

    30% des monuments sans adresse précise. Solution : script Python de géocodage via OpenStreetMap.

    Résultats et Bilan

    Après 3 mois de développement :

    • 8 000+ monuments intégrés avec fiches complètes
    • Système de réservation d'événements culturels opérationnel
    • Système de gestion de compte personnel sécurisé

    Principaux apprentissages : gestion d'équipe pluridisciplinaire, méthodologie de projet agile, et résilience face aux imprévus techniques. Le projet a reçu les félicitations du jury pour son innovation dans la valorisation du patrimoine.
